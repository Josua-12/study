1. 협업과 브랜치
    1) GitHub 원격 저장소 활용
        - 한 장소에서 여러 사라밍 함께 작업 가능
            - 브랜치 기능 덕분임
                - main 브랜치와 구분되는 작업을 하기 위해 코드를 분리하는 것(분기)
    2) 협업 과정
        - 1단계 : 협업을 위한 저장소를 만듦
            ex) 3팀이 있다고 하면 각 팀이 사용할 리파지토리를 만들어야 함
        - 2단계 : 팀원(공동 작업자)을 추가함
            ex) 리파지토리의 권한이 부여된 팀원
        - 3단계 : 브랜치별로 커밋하고 푸시함
        - 4단계 : pull request를 요청함
        - 5단계 : 코드 리뷰 & 병합
    3) 2단계
        - 프로젝트 관리하기
            - 새로은 프로젝트를 생성
            - 이슈 등록
