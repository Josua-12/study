1. 플랫폼으로서의 웹 (일상적으로 접하는 웹)
    1) 디지털 트랜스폼
        - 오프라인에 있는 모든 정보와 서비스를 온라인으로 옮기는 작업

2. 네트워크의 네크워크 (컴퓨터간의 연결. cf. 네트워크:웹은 로컬에만 있으면 의미가 없음)
    1) 라우터 (router. 중간에 연결해주는 역할)
    2) 서버와 클라이언트 (정보를 제공하고 받는 쪽. 한쪽이 둘 다 할 수 있음)
    
3. 프론트엔드 개발과 백엔드 개발
    1) 프론트엔드(front end)
        - 사용자 앞에 보이는 영역을 의미
        - 웹 브라우저(주된 수단) 화면에 보이는 것
        - HTML, CSS, JavaScript
    2) 백엔드(back end)
        - 사용자 입장에서 보이지 않는 영역을 의미
        - 서버에 데이터베이스 형태로 정보는 저장됨
        - 자바, 노드

4. 웹 기본 동작
    1) IP 주소와 도메인, DNS 서버(맵핑)
                              도메인(google.com 등에 입력)         ip 주소
        - 사용자(브라우저 사용) <----------------------> DNS 서버 <------>  서버
    2) HTTP(S) 프로토콜
        - HTTP : 텍스트 데이터 전송
        - HTTPS : 암호화된 데이터 전송

5. 웹 접근성
    1) 나이, 장애 등에 상관없이 모든 사용자에게 접근할 수 있어야 해야한다. 
        ex)시각장애인에게 화면 낭독기로 웹을 이용할 수 있어야 한다.
