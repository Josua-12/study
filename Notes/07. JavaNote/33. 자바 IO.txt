1. 스트림 (stream)
    1) 자료가 오고갈 수 있는 가상의 데이터 통로
    2) 네트워크나 파일과 같은 데이터 흐름을 물의 흐름에 비유한 개념
    3) 자료를 읽어오는 입력 소스(source)와, 자료를 출력하는 대상(target)에 따라 각각 다른 스트림 클래스 제공됨

2. 스트림의 종류
    1) 입력 스트림과 출력 스트림
        - 스트림은 단방향으로 데이터가 흐름
        - 입력과 출력은 각각 별도의 스트림으로 처리되며, 동시에 하나의 스트림에서 입출력을 모두 수행할 수는 없음
        - 입력 스트림
            - InputStream (바이트 기반)
            - Reader (문자 기반)
        - 출력 스트림
            - OutputStream (바이트 기반)
            - Writer (문자 기반)
    2) 바이트 단위 스트림과 문자 단위 스트림 
        - 바이트 스트림
            - 자바의 기본 스트림은 원래 바이트(byte) 단위로 데이터 입출력을 처리함
            - 그림, 동영상, 음악 파일처럼 이진 데이터를 다룰 때 사용
            - 대표 클래스 : ~~InputStream, ~~OutputStream
        - 문자 스트림   
            - 문자 하나를 나타내는 char형은 2바이트임
            - 텍스트 파일처럼 문자 데이터를 다룰 때 사용
            - 대표 클래스 : ~~Reader, ~~Writer
    3) 기반 스트림과 보조 스트림
        - 기반 스트림
            - 데이터의 입출력 대상(파일, 키보드 등)에 직접 연결되어 있음
            - 읽고 쓰는 기능을 직접 제공함
        - 보조 스트림   
            - 직접 입출력을 처리하지는 않음
            - 다른 스트림(기반 스트림 등)을 감싸서 기능을 보조하거나 확장함
            
        - 예)
            기반 스트림 + 보조 스트림A + 보조 스트림B
            FileInputStream + BufferedInputStream + DataInputStream

3. 표준 입출력
    1) 표준 출력 스트림 
        - 변수 이름 : out 
        - 자료형 : static PrintStream 
    2) 표준 입력 스트림     
        - 변수 이름 : in 
        - 자료형 : static InputStream 
    3) 표준 오류 출력 스트림    
        - 변수 이름 : err 
        - 자료형 : static OutputStream

4. 바이트 단위 스트림
    1) InputStream  
        - 바이트 단위로 읽는 스트림의 최상위 클래스
        - 추상 메서드를 포함한 추상 클래스 
        - FileInputStream, ByteArrayInputStream, FilterInputStream
        - 메서드    
            - int read()
                - 입력 스트림으로부터 한 바이트의 자료를 읽음
                - 읽은 자료의 바이트 수를 반환함
            - int read(byte[] b)
                - 입력 스트림으로부터 b[] 크기의 자료를 b[]에 읽음
                - 읽은 자료의 바이트 수를 반환함
            - int read(byte[] b, int off, int len) 
                - 입력 스트림으로부터 b[] 크기의 자료를 b[]의 off 변수 위치부터 저장하며 len만큼 읽음
                - 읽은 자료의 바이트 수를 반환함
            - void close()
                - 입력 
    2) OutputStream
        - 바이트 단위로 쓰는 스트림 중 최상위 클래스
        - FileOutputStream, ByteArrayOutputStream, FilterOutputStream
        - void write(int b)
            - 한 바이트를 출력함
        - void write(byte[] b)
            - b[] 배열에 저장된 모든 데이터를 출력함
        - void write(byte[] b, int off, int len)
            - b[] 배열의 off 위치부터 len 개의 데이터를 출력함
        - void flush()
            - 출력 버퍼에 남아있는 데이터를 강제로 출력함
        - void close()
            - 스트림을 닫고, 출력 버퍼에 남아있는 데이터를 비움

5. 문자 단위 스트림
    1) Class Reader
        - 문자 단위로 읽는 스트림 중 최상위 스트림
        - 하위 클래스
            - Class FileReader
            - Class InputStreamReader
            - Class BufferedReader
        - 자료를 읽는 메서드    
            - int read()
                - 파일로부터 한 문자를 읽고 그 값을 반홚마
            - int read(char[] cbuf)
                - 파일로부터 문자 배열 cbuf의 문자를 읽음
            - int read(char[] cbuf, int off, int len)
                - 파일로부터 문자 배열 cbuf의 off 위치에서부터 len 개수만큼 읽음
            
            - void close()
                - 스트림과 연결된 파일 리소스를 닫음
    2) Class Writer 
        - 문자 단위로 출력하는 스트림 중 가장 최상위 스트림
        - 하위 클래스
            - Class FileWriter
            - Class OutputStreamWriter
            - Class BufferedWriter
        - 자료를 쓰는 메서드
            - void write(int c)
                - 한 문자를 파일에 출력함
            - void write(char[] cbuf)
                - 문자 배열 cbuf의 내용을 파일에 출력함
            - void write(char[] cbuf, int off, int len)
                - 문자 배열 cbuf의 off 위치에서부터 len 개수만큼 읽음
            - void write(String str, int off, int len)
                - 문자열 str의 off 위치에서부터 len 개수만큼 읽음

            - void flush()
                - 파일에 출력하기 전에 자료가 있는 공간(출력 버퍼)을 비워 출력함
            - void close()
                - 파일과 연결된 스트림을 닫음
                - 출력 버퍼도 비워짐

6. 보조 스트림  
    1) 다른 스트림에 기능을 추가하는 보조 스트림
    2) 입출력 대상이 되는 파일이나 네트워크에 직접 쓰거나 읽는 기능은 없음
    3) Wrapper 스트림이라고 함
        - 다른 스트림을 감쌈
        - 생성자의 매개변수로 다른 스트림을 받음
    4) FilterInputStream과 FileOutputStream 생성자
        - protected FilterInputStream(InputStream in)
            - 생성자의 매개변수로 InputStream을 받음
        - FilterOutputStream(OutputStream out)
            - 생성자의 매개변수로 OutputStream을 받음
    5) 구성
        - 기반 스트림 + 보조 스트림
        - 바이트 단위 파일 입력 스트림 + 문자로 변환 기능 추가 + 버퍼링 기능 추가
    6) Buffered 스트림
        - BufferedInputStream
            - 바이트 단위로 읽는 스트림에 버퍼링 기능을 제공
        - BufferedOutputStream
            - 바이트 단위로 출력하는 스트림에 버퍼링 기능을 제공
        - BufferedReader
            - 문자 단위로 읽는 스트림에 버퍼링 기능 제공
        - BufferedWriter
            - 문자 단위로 쓰는 스트림에 버퍼링 기능을 제공
